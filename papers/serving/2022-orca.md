# Paper：Orca

## Citation

- Authors：Gyeong-In Yu 等
- Venue / Year：OSDI 2022
- Title：Orca: A Distributed Serving System for Transformer-Based Generative Models
- Link：https://www.usenix.org/conference/osdi22/presentation/yu

## 一句话总结

Orca 提出 iteration-level scheduling 和 selective batching，让生成式 Transformer serving 能在不同请求长度下更高效地 batch。

## 问题

生成式模型的请求长度和 decode 进度不同，传统 request-level batching 会导致等待和资源浪费，难以同时获得高 throughput 和低 latency。

## 关键想法

把调度粒度从完整 request 降到 decode iteration，并允许不同阶段、不同长度的请求选择性地 batch 在一起。

## System / Algorithm

需要重点理解：

- iteration-level scheduling。
- selective batching。
- prefill 与 decode 的计算差异。
- 早期 serving system 如何处理动态长度请求。

## Experiments

- Hardware：GPU serving system。
- Workloads：生成式 Transformer 请求流。
- Baselines：静态 batching / request-level batching serving。
- Metrics：throughput、latency、SLO satisfaction。

## 复现计划

不需要完整复现 Orca 系统，先复现思想：

- 在 vLLM 中观察 continuous batching 的行为。
- 构造不同 prompt length / decode length 的混合 workload。
- 记录不同 request mix 下的 TTFT、TPOT 和吞吐变化。

## 与我的研究主线的关系

Orca 是理解 serving scheduler 的基线。communication bottleneck 是否进入 critical path，和 scheduler 如何组织 prefill / decode / batch 有直接关系。

## Open Questions

- iteration-level scheduling 是否会产生更多 small-message collectives？
- batching 策略如何改变 TP / EP 下 collective 的 message size？
- Scheduler 是否可能帮助 hide communication？
